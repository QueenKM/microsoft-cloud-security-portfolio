# Contributing

Thanks for contributing to this portfolio repository.

## Principles

- Keep documentation clear, recruiter-friendly, and technically accurate.
- Prefer small, focused pull requests over large unrelated changes.
- Write repository content in English unless a file explicitly needs another language.
- Avoid uploading secrets, tenant identifiers, or non-anonymized screenshots.

## Branching

- Use short descriptive branch names such as `feature/cloud-lab-rbac-matrix`.
- Keep one topic per branch when possible.

## Documentation Rules

- Use relative Markdown links inside the repository.
- Add a short "business value" angle when documenting a project change.
- Capture setup assumptions so another person can reproduce the work.
- Keep diagrams and screenshots aligned with the current repo state.

## Validation

Before opening a pull request:

- run Markdown and YAML checks if you changed docs or workflows
- review links and headings
- confirm that screenshots and datasets are safe to publish

## Pull Requests

- Summarize what changed, why it changed, and how it was validated.
- Call out tradeoffs or areas that still need follow-up.
- Default to draft when the work is still being shaped.
