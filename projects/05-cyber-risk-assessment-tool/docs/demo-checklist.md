# Demo Checklist

## Minimum Walkthrough

- [ ] Launch the desktop application.
- [ ] Load the sample ransomware scenario.
- [ ] Explain the difference between inherent and residual risk.
- [ ] Show the 5x5 matrix update after editing one risk.
- [ ] Export the register to `CSV`, `Excel`, and `PDF`.
- [ ] Open the generated files and explain how they support reporting.

## Interview Storyline

1. Start with the practical problem: most university risk exercises stop at a spreadsheet.
2. Show that the app turns the scoring model into a usable desktop workflow.
3. Explain the residual-risk logic and how controls reduce the score.
4. Show that the project is not just UI, because it includes persistence and multi-format export.
5. Close with how the app complements the Azure and governance projects in the same portfolio.
