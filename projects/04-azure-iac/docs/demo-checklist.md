# Demo Checklist

## Minimum Walkthrough

- [ ] Open the project README and explain the landing zone goal.
- [ ] Show the Bicep baseline preview artifact.
- [ ] Show the deployment guide and validation checklist.
- [ ] Explain the resource-group split between monitoring, security operations, and workload.
- [ ] Explain the diagnostic settings and alert modules.
- [ ] Explain how `Project 04` feeds the live `Cloud Security Lab`.

## Interview Storyline

1. Start with repeatability: the security lab is not a click-built environment.
2. Show the baseline design and why the monitoring workspace is created centrally.
3. Explain how diagnostics and scheduled query alerts are deployed as part of the baseline.
4. Point out that optional VM and RBAC assignments are parameter-driven, not hardcoded.
5. Finish with how this project supports `Project 01` and proves infrastructure discipline.
