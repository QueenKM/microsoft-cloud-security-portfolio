# Demo Checklist

## Goal

Use this checklist when presenting the toolkit in GitHub, an interview, or a university demonstration.

## Minimum Walkthrough

- [ ] Open the project README and explain the business problem.
- [ ] Show the governance snapshot table.
- [ ] Show the guest access decision tree.
- [ ] Show the policy pack and explain the difference between guests and shared channels.
- [ ] Show the owner quick-start guide.
- [ ] Show the lifecycle checklist and exception process.
- [ ] Show the team request template and quarterly review template.

## Interview Storyline

1. Explain that uncontrolled Teams growth creates naming, access, and lifecycle risk.
2. Show that the toolkit uses policy plus operations, not policy only.
3. Explain why shared channels reduce oversharing in some external collaboration cases.
4. Show how ownership, lifecycle reviews, and archive-before-delete reduce stale workspaces.
5. Close with how this complements the Azure and Zero Trust projects in the portfolio.

## Strong Evidence To Add Later
