# Lifecycle Checklist

## Stage 1: Request

- [ ] business purpose defined
- [ ] department or sponsor identified
- [ ] primary owner assigned
- [ ] backup owner assigned
- [ ] guest or shared channel need declared
- [ ] sensitivity expectation declared
- [ ] intended lifespan estimated

## Stage 2: Launch

- [ ] naming standard applied
- [ ] correct privacy setting applied
- [ ] sensitivity label applied if required
- [ ] initial members added
- [ ] guest access approved if used
- [ ] owner quick-start guide shared

## Stage 3: Operate

- [ ] owners remain active
- [ ] membership remains current
- [ ] guests remain justified
- [ ] channel structure remains usable and not duplicated elsewhere
- [ ] content scope still matches the original purpose

## Stage 4: Review

Run at least every 180 days.

- [ ] team still has a valid business purpose
- [ ] at least two owners remain assigned
- [ ] guests were reviewed
- [ ] shared channels were reviewed
- [ ] retention or recordkeeping needs remain understood
- [ ] decision captured: renew, archive, delete, or exception

## Stage 5: Archive

Archive when the team is inactive but still needs historical reference.

- [ ] owners confirmed the work is complete
- [ ] guests removed where possible
- [ ] open tasks or files moved if needed
- [ ] archive decision logged
- [ ] archive date recorded

## Stage 6: Delete

Delete only after archive or when immediate cleanup is justified.

- [ ] business sponsor approved deletion
- [ ] compliance concerns checked
- [ ] no open exception remains
- [ ] deletion date recorded

## Recommended Timing

| Team Type | First Review | Standard Renewal Pattern |
| --- | --- | --- |
| Project team | 90 days | Every 180 days until closed |
| Department team | 180 days | Every 180 days |
| Vendor or partner team | 90 days | Every 90 to 180 days depending on external access risk |
| Event or short-term cohort team | 30 to 60 days after event | Archive quickly, then delete if no retention need exists |
