# Implementation Checklist

## Goal

Use this checklist to move from policy design to tenant implementation in a controlled sequence.

## Phase 0: Prepare

- [ ] Confirm governance sponsor and service owner.
- [ ] Confirm core platforms in scope: Teams, Entra ID, SharePoint, Purview.
- [ ] Confirm current tenant creation model: open self-service or controlled intake.
- [ ] Confirm whether shared channels are in scope for external collaboration.
- [ ] Confirm whether naming policy, sensitivity labels, lifecycle, and retention features are available in the tenant.

## Phase 1: Define Baseline Rules

- [ ] Approve naming convention and blocked-word list.
- [ ] Approve minimum owner model of two owners per team.
- [ ] Approve default guest access stance.
- [ ] Approve when shared channels are preferred over guests.
- [ ] Approve team classes that require sensitivity labels.
- [ ] Approve lifecycle checkpoints and archive-before-delete rule.

## Phase 2: Configure Tenant Controls

- [ ] Configure Microsoft 365 group naming policy.
- [ ] Configure Teams guest access settings.
- [ ] Configure channel policy for shared channel creation and external participation.
- [ ] Configure sensitivity labels for relevant team classes.
- [ ] Configure Microsoft 365 group expiration policy if adopted.
- [ ] Configure Purview retention policies for Teams chat and channel content.

## Phase 3: Operationalize Intake

- [ ] Publish the team request template.
- [ ] Publish the owner quick-start guide.
- [ ] Publish the quarterly review template.
- [ ] Decide where requests and exception records are stored.
- [ ] Define response-time targets for standard and exception requests.

## Phase 4: Launch

- [ ] Pilot the toolkit with one low-risk department and one cross-functional project team.
- [ ] Test a guest access request.
- [ ] Test a shared channel request.
- [ ] Test lifecycle renewal messaging with a pilot owner.
- [ ] Capture lessons learned and update policy wording.

## Phase 5: Operate

- [ ] Review teams without two owners every month.
- [ ] Review guest-heavy teams quarterly.
- [ ] Review archived teams pending deletion.
- [ ] Review exceptions before expiry.
- [ ] Review policy pack every six months against Microsoft roadmap and institutional needs.

## Deliverables At Completion

- approved policy pack
- operating model
- owner guide
- request and review templates
- lifecycle checklist
- exception register process
