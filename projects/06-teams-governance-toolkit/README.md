# Project 06: Microsoft Teams Governance Toolkit

## Goal

Create a practical governance repository for Microsoft Teams that defines collaboration guardrails without blocking the business.

## Why This Project Matters

This project rounds out the portfolio with Microsoft 365 governance depth and shows that you understand how policy, security, and user adoption fit together.

## Core Scope

- naming conventions
- team lifecycle guidance
- guest access policy
- retention considerations
- sensitivity and classification guidance
- ownership and approval model

## Deliverables

- governance policy pack
- implementation checklist
- onboarding templates
- owner responsibilities guide
- review and exception process

## Build Phases

1. Define governance objectives and user groups.
2. Draft policies for naming, access, retention, and lifecycle.
3. Add templates and checklists for administrators and owners.
4. Map policies to risk scenarios and compliance concerns.
5. Package the toolkit for GitHub presentation.

## Evidence To Capture

- policy tables
- lifecycle checklist
- guest access decision tree
- owner quick-start guide
