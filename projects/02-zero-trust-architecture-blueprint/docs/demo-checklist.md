# Demo Checklist

## Goal

Provide a short, repeatable walkthrough for presenting the Zero Trust blueprint in a portfolio review or interview.

## Before The Demo

- open the reference diagram
- open the identity and device flow diagram
- open the policy pack
- open one decision log
- open the STRIDE threat model

## Demo Sequence

1. Explain the `Astera University` scenario in under one minute.
2. Define the three Zero Trust principles.
3. Walk through the trust boundaries on the reference diagram.
4. Show how a privileged admin request is evaluated across identity, device, and application signals.
5. Highlight two concrete policy decisions and their tradeoffs.
6. Finish with the STRIDE model to show how risks were analyzed, not guessed.

## Key Talking Points

- identity is the control plane, not the network perimeter
- privileged work requires stronger controls than collaboration work
- device trust matters for sensitive access
- least privilege must be visible in both policy and role design
- centralized telemetry is required for Zero Trust to be defendable

## Evidence To Add Later

- exported diagrams
- policy screenshots from a real tenant
- one mapped incident or alert example
